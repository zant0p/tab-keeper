🔒 Tab Keeper Privacy Policy

Last Updated: April 9, 2026
Extension Version: 1.0.0

🛡️ Key Point: Tab Keeper operates entirely locally within your browser. No data is transmitted to external servers, third parties, or the developer.

Overview

Tab Keeper is a Chrome extension that helps you maintain focus on a specific tab by automatically switching back after a configurable period of inactivity. This privacy policy explains what data the extension collects, how it's used, and your rights.

Data Collection & Use

What Data We Access

Data Type Purpose Storage Location
Target URL The URL to keep active Chrome local storage
Return Timer Inactivity timeout (1-60 min) Chrome local storage
Username/Password Auto-login credentials (optional) Chrome local storage
Active Tab Status Detect when user switches tabs In-memory only (not stored)
Current URL Detect login pages for auto-fill In-memory only (not stored)

What We DO NOT Collect

Tab Keeper explicitly does NOT:

  • ❌ Collect or store browsing history
  • ❌ Transmit any data to external servers
  • ❌ Use analytics or tracking services
  • ❌ Sync data to Google accounts or cloud services
  • ❌ Share data with third parties (including advertisers)
  • ❌ Sell or rent user data
  • ❌ Access data from other websites or extensions
  • ❌ Mine cryptocurrency
  • ❌ Display ads

Limited Use Compliance

✅ Tab Keeper complies with the Chrome Web Store User Data Policy, including the Limited Use requirements.

Specifically:

  1. Single Purpose: Data is used solely for the extension's core functionality (tab monitoring and auto-login)
  2. No Third-Party Transfers: User data is never transferred to any third party under any circumstances
  3. No Human Access: No human (including the developer) reads or accesses user data
  4. No Advertising: Data is not used for personalized ads or ad targeting
  5. No Credit/Lending: Data is not used for credit-worthiness or lending decisions

Data Security

How We Protect Your Data

  • Local Storage Only: All data is stored in Chrome's chrome.storage.local API
  • Extension Isolation: Data is accessible only by the Tab Keeper extension
  • Profile-Bound: Data is tied to your Chrome browser profile
  • No Transmission: Data never leaves your device
  • No Encryption: Chrome local storage is not encrypted (plain text)
⚠️ Important Security Notes:
  • Avoid Sensitive Accounts: Do not use auto-login with banking, financial, or highly sensitive accounts
  • Dedicated Profile: Consider using a dedicated Chrome profile for kiosk/monitoring use
  • Clear Credentials: Remove stored credentials when no longer needed (via extension settings)
  • Strong Passwords: Use strong, unique passwords for monitored accounts
  • Shared Devices: Do not use on shared/public computers without clearing data after use

Permissions

Tab Keeper requests the following Chrome permissions:

Permission Why It's Needed
tabs Monitor active tab and switch to target URL
storage Save settings and credentials locally
alarms Periodic background checks for inactivity
activeTab Access current tab for login detection
scripting Inject auto-fill script on login pages
<all_urls> Detect login forms on any website

We request only the minimum permissions necessary. No permission is used for purposes other than the extension's core functionality.

User Rights & Choices

You Control Your Data

  • View Data: All stored data is visible in the extension's Options page
  • Edit Data: You can modify or clear any stored data at any time
  • Delete Data: Uninstalling the extension removes all stored data
  • Opt-Out: Disable the extension or clear credentials via Options page

Data Retention

Data is retained indefinitely until you:

  • Clear it manually via Options page
  • Uninstall the extension
  • Clear Chrome's browsing data

Children's Privacy

Tab Keeper is not intended for users under 13 years of age. We do not knowingly collect data from children under 13. If you are a parent or guardian and believe your child has provided data through this extension, please contact us.
